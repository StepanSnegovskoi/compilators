# Транслятор и виртуальная машина языка Pascal: Техническое руководство

Данный проект представляет собой полноценный компилятор подмножества языка Pascal, разработанный на языке Python. Проект включает в себя полный цикл трансляции исходного кода: от чтения текстовых символов до выполнения в собственной виртуальной среде. Все компоненты реализованы вручную методом рекурсивного спуска без использования сторонних генераторов парсеров.

---

## 1. Обзор архитектуры
Система спроектирована как модульный конвейер, где каждый блок отвечает за строго определенную фазу обработки кода:

| Модуль | Ответственность |
| :--- | :--- |
| `main.py` | Точка входа: чтение файлов, запуск конвейера, обработка ошибок. |
| `lexer.py` | Лексический анализ: токенизация входной строки. |
| `parser.py` | Синтаксический анализ: построение AST методом рекурсивного спуска. |
| `nodes.py` | Библиотека классов AST: иерархическая структура данных. |
| `semantic.py` | Семантический анализатор: проверка типов и областей видимости. |
| `optimizer.py` | Оптимизатор AST: свертка констант и упрощение дерева. |
| `interpreter.py` | Виртуальная машина: непосредственное выполнение кода. |

---

## 2. Подробное описание компонентов

### 2.1. Лексический анализатор (`Lexer`)
Лексер является первым звеном конвейера. Его задача — преобразовать поток символов в поток токенов.

* **Класс `Token`**: Хранит тип токена (`KEYWORD`, `NAME`, `NUMBER`, `STRING`, `OPERATOR`, `PUNCT`), значение и метаданные (строка, колонка).
* **Класс `Lexer`**: Управляет курсором `pos` в строке кода. 
    * **Пропуск мусора**: Игнорирует пробелы, табуляции и переводы строк (с обновлением счетчиков `line` и `column`).
    * **Комментарии**: Реализована поддержка однострочных комментариев, которые игнорируются сканером.
    * **Числа**: Реализован сбор цифр в числовой тип.
    * **Составные операторы**: Лексер реализует lookahead (просмотр вперед), что позволяет корректно отличать `:` от `:=` или `<` от `<=`.

### 2.2. Синтаксический анализатор (`Parser`)
Парсер реализует грамматику Pascal, преобразуя плоский список токенов в древовидную структуру (AST).

* **Метод рекурсивного спуска**: Каждая конструкция языка (программа, объявление переменных, циклы, выражения) представлена отдельным методом.
* **Обработка приоритетов**: Реализована классическая цепочка вызовов (от логических операций к арифметическим), что гарантирует корректный порядок вычислений (`2 + 3 * 4` будет обработано верно).
* **Класс `Parser`**: Управляет процессом разбора, используя методы `match` и `is_match` для проверки ожидаемых токенов. При обнаружении несоответствия грамматике генерируется `SyntaxError` с точным указанием позиции.

### 2.3. Структура узлов (`nodes.py`)
Здесь описаны классы для каждого узла дерева.
* **`Node` (Базовый)**: Хранит метаданные (строка/колонка).
* **`PascalVisitor`**: Реализует паттерн «Посетитель» (Visitor), который критически важен для расширяемости проекта. Любой модуль (семантика, оптимизатор, интерпретатор) наследуется от этого класса, чтобы обходить дерево, не меняя структуру самих узлов.

### 2.4. Семантический анализатор (`semantic.py`)
Этот модуль отвечает за проверку корректности программы до начала её выполнения.

* **`SymbolTable`**: Реализует дерево областей видимости. Каждая область видимости имеет родительскую ссылку. При поиске переменной поиск идет снизу вверх (локальная -> глобальная).
* **Проверка типов**: В процессе обхода AST анализатор высчитывает типы узлов. Например, при сложении проверяется, чтобы оба операнда были числами.
* **Проверка сигнатур**: При объявлении функции запоминается количество параметров. При вызове — происходит сверка.
* **Сборщик ошибок**: Реализован механизм накопления ошибок в список, а не прерывание при первом сбое. Это позволяет пользователю увидеть весь список опечаток за один запуск.

### 2.5. Оптимизатор (`optimizer.py`)
Реализует предобработку AST-дерева.
* **Constant Folding**: Если узел `BinOpNode` видит два дочерних `NumberNode`, он схлопывает их в один узел с результатом операции. 
* **Преимущества**: Снижает нагрузку на CPU при выполнении программы внутри интенсивных циклов, так как вычисления выполняются один раз на этапе трансляции.

### 2.6. Интерпретатор (`Interpreter`)
Виртуальная машина, исполняющая AST-дерево.

* **`Env` (Окружение)**: Хранилище переменных в текущем контексте исполнения. Содержит словарь связей и ссылку на внешнее окружение.
* **Логика выполнения**:
    * `visitAssignNode`: Вычисляет выражение справа и сохраняет в переменную.
    * `visitWhileNode` / `visitForNode`: Используют механизмы Python для организации циклов.
    * `visitCallNode`: Реализует системные вызовы. `Write` транслируется в `print`, `Read` — в `input`. При вызове процедур создается новое окружение `Env`, имитирующее стек вызовов.

## 3. Технические аспекты реализации

### Управление памятью
В данной реализации память управляется через контексты исполнения (`Env`). При входе в процедуру выделяется новый контекст, который автоматически удаляется после возврата (выхода из области видимости), что обеспечивает корректную работу локальных переменных.

### Обработка ошибок
Система ошибок построена на перехвате специфических исключений. Модуль `main.py` использует `try...except` для перехвата `SystemExit`, что позволяет компилятору продолжать обработку следующего файла, даже если текущий содержит ошибки.

### Выполнение
Интерпретатор использует рекурсивный обход узлов. Для ускорения доступа к переменным используется поиск в цепочке словарей (окружений). При вычислении выражений интерпретатор опирается на динамическую типизацию Python, что упрощает написание, при этом семантический анализатор гарантирует, что типы соответствуют правилам языка.

---

## 4. Как это использовать
1. Создайте папку `examples/` в корне проекта.
2. Поместите туда файлы с кодом на языке Pascal (расширение `.pas`).
3. Запустите интерпретатор командой: `python main.py`.
4. Результат будет выведен в консоль вместе с визуализацией структуры AST для каждого файла.
