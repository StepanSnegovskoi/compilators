# Интерпретатор языка Pascal: Техническое руководство

Данный проект представляет собой интерпретатор подмножества языка Pascal, разработанный на языке Python. Проект включает полный цикл обработки исходного кода: от лексического анализа до непосредственного выполнения AST-дерева. Грамматика реализована методом рекурсивного спуска.

## 1. Обзор архитектуры

Система спроектирована как модульный конвейер:

| Модуль | Ответственность |
| --- | --- |
| main.py | Точка входа: чтение файлов, запуск конвейера |
| lexer.py | Лексический анализ: токенизация |
| parser.py | Синтаксический анализ: построение AST |
| nodes.py | Классы узлов AST |
| semantic.py | Семантический анализ: типы и области видимости |
| optimizer.py | Оптимизация AST: свертка констант |
| interpreter.py | Интерпретатор: выполнение программы |
| errors.py | Обработка ошибок |
| print_ast.py | Визуализация AST |

## 2. Подробное описание компонентов

### 2.1. Лексический анализатор (lexer.py)

Лексер преобразует поток символов в поток токенов.

Класс Token (в token.py): Хранит тип токена (KEYWORD, NAME, NUMBER, STRING, OPERATOR, PUNCT), значение, строку и колонку.

Класс Lexer:
- Управляет курсором в строке кода
- Пропускает пробелы, табуляции, переводы строк
- Поддерживает комментарии в фигурных скобках { ... }
- Собирает числа, идентификаторы, ключевые слова
- Обрабатывает строки в одинарных кавычках
- Использует просмотр вперед для различения : и :=, < и <=, . и ..
- Обрабатывает символы: +, -, *, /, (, ), [, ], ;, ,, .

### 2.2. Синтаксический анализатор (parser.py)

Парсер реализует грамматику Pascal методом рекурсивного спуска.

Поддерживаемые конструкции:
- Программа: program имя; блок .
- Объявление переменных: var список : тип;
- Объявление массивов: array [нижняя .. верхняя] of тип
- Объявление процедур: procedure имя [(параметры)]; блок ;
- Составной оператор: begin операторы end
- Присваивание: переменная := выражение
- Условный оператор: if условие then оператор [else оператор]
- Цикл while: while условие do оператор
- Цикл do-while: do оператор while условие
- Цикл for: for переменная := старт to финиш do оператор
- Вызов процедур: имя [(аргументы)]

Приоритет операций (от низших к высшим):
1. or
2. and
3. сравнения (=, <>, <, >, <=, >=)
4. сложение/вычитание (+, -)
5. умножение/деление (*, /, div, mod)
6. унарные (+, -, not)

### 2.3. Структура узлов (nodes.py)

Базовый класс Node хранит строку и колонку.

| Класс | Назначение |
| --- | --- |
| ProgNode | Корень программы |
| BlockNode | Блок (объявления + операторы) |
| VarDeclNode | Объявление переменной |
| ProcDeclNode | Объявление процедуры |
| AssignNode | Присваивание |
| IfNode | Условный оператор |
| WhileNode | Цикл с предусловием |
| DoWhileNode | Цикл с постусловием |
| ForNode | Цикл со счетчиком |
| CallNode | Вызов процедуры |
| BinOpNode | Бинарная операция |
| UnaryOpNode | Унарная операция |
| VarNode | Переменная |
| ArrayAccessNode | Доступ к элементу массива |
| NumberNode | Числовая константа |
| CharNode | Символьная константа |
| BoolNode | Логическая константа |
| EmptyNode | Пустой оператор |

Класс PascalVisitor: Реализует паттерн «Посетитель». Все модули (семантика, оптимизация, интерпретация) наследуются от него для обхода дерева.

### 2.4. Семантический анализатор (semantic.py)

Класс SymbolTable:
- Реализует вложенные области видимости
- Методы define(name, type, info) и lookup(name) (поиск снизу вверх)

Класс SemanticAnalyzer (наследник PascalVisitor):
- Проверяет объявление переменных перед использованием
- Проверяет совместимость типов при присваивании
- Проверяет, что условие в if и while имеет тип boolean
- Проверяет, что индекс массива — целое число
- Проверяет количество аргументов при вызове процедур
- Встроенные процедуры предопределены: Write, WriteLn, Read, ReadLn, Inc, Dec, Abs

Обработка ошибок: При первой семантической ошибке выполнение файла прекращается.

### 2.5. Оптимизатор (optimizer.py)

Реализует свертку констант (Constant Folding).

Принцип: Если узел бинарной операции имеет два дочерних узла-числа, он заменяется на один узел-число с вычисленным значением.

Пример: 5 + 3 * 2 -> после оптимизации 11

Поддерживаемые операции: +, -, *, div.

### 2.6. Интерпретатор (interpreter.py)

Класс Env (окружение):
- b — словарь связей имя -> значение
- p — ссылка на родительское окружение
- get(name) и set(name, value) с поиском по цепочке

Класс Interpreter (наследник PascalVisitor):
- Создает глобальное окружение
- Хранит объявления процедур в словаре procs
- При вызове процедуры создает новое окружение, связывает параметры с аргументами

Поддерживаемые операции:
- Арифметика: +, -, *, div, mod
- Сравнения: =, <>, <, >, <=, >=
- Логика: and, or, not
- Ввод/вывод: Read/ReadLn (чтение числа), Write/WriteLn (вывод)
- Массивы: обращение по индексу, присваивание элементам

### 2.7. Обработка ошибок (errors.py)

Глобальный объект errors:
- has_error — флаг наличия ошибки
- error(message, line, column) — выводит сообщение и вызывает sys.exit(1)

## 3. Поддерживаемое подмножество Pascal

### 3.1. Типы данных
- integer — целые числа
- boolean — true / false
- char — символы в одинарных кавычках
- array[нижняя .. верхняя] of тип — статические массивы

### 3.2. Управляющие конструкции
- if условие then оператор [else оператор]
- while условие do оператор
- do оператор while условие
- for переменная := старт to финиш do оператор
- begin ... end — составной оператор

### 3.3. Процедуры
- Объявление: procedure имя(параметры); begin ... end;
- Вызов: имя(аргументы)
- Параметры передаются по значению

### 3.4. Встроенные процедуры и функции
- Write(x) / WriteLn(x) — вывод
- Read(x) / ReadLn(x) — ввод числа
- Inc(x) / Dec(x) — увеличение/уменьшение на 1
- Abs(x) — абсолютное значение

### 3.5. Комментарии
- Многострочные: { комментарий }

## 4. Запуск интерпретатора

### 4.1. Развертывание проекта и подготовка окружения

**Системные требования:**
* **Интерпретатор:** Python версии 3.7 или выше.
* **Операционная система:** Windows, macOS или Linux.
* **Сторонние зависимости:** Отсутствуют.

**Пошаговое руководство по настройке среды (с нуля):**

1. **Скачивание исходного кода:** Склонируйте репозиторий с GitHub с помощью команды `git clone` или скачайте проект в виде ZIP-архива и распакуйте его в рабочую директорию на вашем компьютере.
   
2. **Проверка состава модулей:**
   Убедитесь, что все необходимые файлы транслятора находятся в одной корневой папке. Локальный импорт модулей завязан на относительные пути, поэтому в каталоге должны присутствовать:
   * `main.py` (главный скрипт)
   * `lexer.py` (лексический анализатор)
   * `parser.py` (синтаксический парсер)
   * `nodes.py` (структура классов AST)
   * `semantic.py` (контроль типов и таблица символов)
   * `optimizer.py` (оптимизация дерева)
   * `interpreter.py` (виртуальная машина исполнения)
   * `errors.py` (модуль перехвата ошибок)
   * `print_ast.py` (визуализатор дерева)
   * `token.py` (представление токена)

3. **Изоляция и внешние библиотеки (Zero-Dependency):**
   Вам **не требуется** создавать виртуальное окружение (`python -m venv .venv`), активировать его и устанавливать внешние пакеты через менеджер `pip`. Проект спроектирован по принципу *Zero-Dependency* — лексер и парсер рекурсивного спуска написаны вручную, поэтому код работает исключительно на базе встроенных возможностей стандартной библиотеки Python.

4. **Подготовка директории тестов:**
   В корневой папке проекта (рядом с `main.py`) создайте подкаталог с именем `examples`, если его не существует и положите туда файлы с кодом Pascal, .pas:
   ```bash
   mkdir examples
   ```
5. **Запуск**: Выполните команду в терминале:
   ```bash
   python main.py
   ```

## 5. Схема работы

Исходный код (.pas)
        |
    lexer.py -> токены
        |
    parser.py -> AST (рекурсивный спуск)
        |
    semantic.py -> проверка типов
        |
    optimizer.py -> свертка констант
        |
    interpreter.py -> выполнение
        |
    Вывод результата
